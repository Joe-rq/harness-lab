# REQ-2026-051: Phase 5.5 部署守卫

## 状态
- 当前状态：completed
- 当前阶段：qa

## 背景
Phase 5.2-5.4 赋予了 AI 自恢复能力：停滞时自动拆分、循环时记录根因、R3+ 时允许（autonomous）。但自恢复能力越强，AI 可能尝试越激进的手段——`rm -rf` 清理、`git push --force` 覆写、`git reset --hard` 丢弃改动。当前 risk-tracker 是 PostToolUse hook，无法真正阻断操作。supervised 模式的"强警告"只是 stderr 提醒，操作已经执行完毕。

## 目标
- 新建 `scripts/deploy-guard.mjs`：PreToolUse Bash hook，拦截危险 Bash 命令
- 注册到 `settings.local.json` PreToolUse Bash 阶段
- 三模式行为分化：collaborative=提醒，supervised=阻断，autonomous=阻断

## 非目标
- 不做生产环境部署拦截（本仓库无部署流程）
- 不拦截非 Bash 工具的危险操作（Write/Edit 已有 scope-guard）
