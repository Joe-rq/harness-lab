# REQ-2026-048: Phase 5.2 自恢复指令

## 状态
- 当前状态：completed
- 当前阶段：qa

## 背景
Phase 5.1 Watchdog 已实现停滞和循环检测，但检测到问题时只注入"你停滞了"之类的提醒，AI 收到后行为不确定——可能忽略、可能重复尝试、可能换方向但没有标准流程。需要为 AI 定义标准响应协议，让检测到异常后有可预测的恢复路径。

**核心约束**：Claude Code hook 架构只支持 inject（additionalContext）和 block（decision），不支持执行动作。所以恢复逻辑必须在 CLAUDE.md（prompt 层面）定义，而非新脚本。

## 目标
- CLAUDE.md 新增"异常响应协议"章节：定义停滞/循环/R3+ 风险的标准响应流程
- 新增 `/resume` slash command：读取 progress.txt + watchdog-state → 建议下一步行动
- 不新增脚本——本质是 prompt 层面的决策指南

## 非目标
- 不修改 hook 脚本（watchdog/loop-detection/risk-tracker 的 inject 内容不变）
- 不实现 autonomous 模式的行为分化（属于 5.4）
- 不实现续传协议（属于 5.3）
