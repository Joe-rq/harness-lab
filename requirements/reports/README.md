# Reports

> 本目录存放与 REQ 绑定的执行报告。
> 报告是仓库内的证据链，不应只停留在聊天记录里。

## 文件命名

- `REQ-YYYY-NNN-code-review.md`
- `REQ-YYYY-NNN-qa.md`
- `REQ-YYYY-NNN-ship.md`
- 需要补充其他报告时，沿用 `REQ-YYYY-NNN-<report-type>.md`

## 报告最低要求

每份报告至少应包含：
- 关联 REQ
- 报告日期
- 执行人 / agent
- 输入材料
- 实际执行的命令或操作
- 结果与发现
- 是否阻塞发布 / 合并
- 后续动作

## 特别约束

- 没有真实执行命令或操作时，不能写“已验证通过”
- 可以写“未执行”，但必须说明原因和风险
- 结论要可追溯到具体文件、命令、日志或截图

## 建议分工

- Code Review：关注正确性、风险、回归和测试缺口
- QA：关注验收标准、功能验证、回归验证和环境说明
- Ship：关注发布条件、发布结果、监控和回滚
