# Skill: /qa

## 🎯 目标
进行质量验证，确保功能正确。

## 📋 执行清单

### 1. 功能验证
- [ ] 按照需求文档验证功能
- [ ] 验证正常流程
- [ ] 验证异常流程
- [ ] 验证边界条件

### 2. 浏览器测试（如适用）
- [ ] 启动应用
- [ ] 操作 UI 流程
- [ ] 截图对比
- [ ] 验证响应

### 3. 日志检查
- [ ] 检查错误日志
- [ ] 检查性能日志
- [ ] 验证关键日志输出

### 4. 回归测试
- [ ] 运行全量测试
- [ ] 检查是否有回归
- [ ] 验证核心流程

## 📝 输出模板

```markdown
## QA Report: [需求/功能]

### 测试环境
- 分支：
- 版本：
- 环境：

### 功能验证
| 测试项 | 状态 | 备注 |
|-------|------|------|
| 功能 A | ✅/❌ | |
| 功能 B | ✅/❌ | |

### 浏览器验证
- 截图：[附截图]
- 结果：✅/❌

### 问题列表
1. 问题描述 → 严重程度 → 建议

### 结论
- [ ] 通过，可以发布
- [ ] 需要修复后重新验证
```

## ⚠️ 红线
- 不验证通过不能标记完成
- 不能只听信"看起来没问题"
- 异常场景必须测试
