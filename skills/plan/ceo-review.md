# Skill: /plan-ceo-review

## 🎯 目标
从产品和业务角度审查需求，确保价值对齐。

## 📋 执行清单

### 1. 需求理解
- [ ] 读取 requirements/INDEX.md 了解需求背景
- [ ] 读取 context/business/ 相关业务文档
- [ ] 确认用户价值和业务目标

### 2. 价值评估
- [ ] 评估需求的用户价值
- [ ] 评估技术可行性
- [ ] 评估资源投入和优先级

### 3. 范围界定
- [ ] 明确需求边界（包含/不包含）
- [ ] 识别关键依赖和阻塞点
- [ ] 定义验收标准

### 4. 输出产物
- [ ] 需求价值评估报告
- [ ] 范围确认文档
- [ ] 风险点提示

## 📝 输出模板

```markdown
## CEO Review: [需求名称]

### 用户价值
- 解决什么问题
- 目标用户是谁
- 预期效果

### 业务目标
- 对齐的产品战略
- 关键指标影响

### 范围确认
**包含**：
- 功能 A
- 功能 B

**不包含**：
- 功能 C（原因）

### 验收标准
1. 标准 1
2. 标准 2

### 风险提示
- 风险 1：描述 + 缓解方案
- 风险 2：描述 + 缓解方案

### 决策建议
- 建议 proceed / 建议暂缓 / 建议调整范围
- 理由
```

## ⚠️ 红线
- 不要在没有明确价值的情况下 proceed
- 不要忽视关键依赖风险
- 范围变更必须重新 Review
