# Skill: /review

## 🎯 目标
进行代码审查，确保质量和规范。

## 📋 执行清单

### 1. 架构合规性
- [ ] 检查分层架构（Types→Config→Repo→Service→UI）
- [ ] 检查依赖方向
- [ ] 检查模块边界

### 2. 代码质量
- [ ] 检查代码可读性
- [ ] 检查命名规范
- [ ] 检查注释完整性
- [ ] 检查复杂度

### 3. 功能正确性
- [ ] 检查业务逻辑
- [ ] 检查边界条件
- [ ] 检查错误处理
- [ ] 检查并发安全

### 4. 测试覆盖
- [ ] 检查单元测试
- [ ] 检查集成测试
- [ ] 检查边界测试
- [ ] 检查异常测试

### 5. 安全审查
- [ ] 检查输入验证
- [ ] 检查 SQL 注入
- [ ] 检查 XSS
- [ ] 检查敏感信息泄露

### 6. 性能检查
- [ ] 检查算法复杂度
- [ ] 检查资源使用
- [ ] 检查缓存使用
- [ ] 检查数据库查询

## 📝 输出模板

```markdown
## Code Review: [PR/变更]

### 审查结果
- [ ] 通过
- [ ] 需要修改
- [ ] 需要讨论

### 发现的问题

#### 严重（必须修复）
1. 问题描述 → 建议方案

#### 中等（建议修复）
1. 问题描述 → 建议方案

#### 轻微（可选）
1. 问题描述 → 建议方案

### 正面反馈
- 做得好的地方

### 行动项
- [ ] 修复问题 1
- [ ] 修复问题 2
- [ ] 更新文档
```

## ⚠️ 红线
- 不通过 Linter 的代码不能合并
- 没有测试的代码不能合并
- 架构违规必须修复
