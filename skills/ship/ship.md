# Skill: /ship

## 🎯 目标
完成发布工程，安全上线。

## 📋 执行清单

### 1. 发布前检查
- [ ] 所有测试通过
- [ ] 代码审查通过
- [ ] 文档已更新
- [ ] 变更日志已更新

### 2. 构建
- [ ] 执行构建
- [ ] 验证构建产物
- [ ] 检查构建日志

### 3. 测试环境验证
- [ ] 部署到测试环境
- [ ] 进行冒烟测试
- [ ] 验证关键功能

### 4. 生产发布
- [ ] 执行发布计划
- [ ] 监控发布过程
- [ ] 验证发布成功

### 5. 发布后检查
- [ ] 监控系统指标
- [ ] 检查错误率
- [ ] 验证用户反馈

## 📝 输出模板

```markdown
## Ship Report: [版本/功能]

### 发布信息
- 版本：
- 时间：
- 发布人：

### 变更内容
- 功能 1
- 功能 2
- 修复 1

### 发布检查
- [ ] 测试通过
- [ ] 审查通过
- [ ] 文档更新

### 发布结果
- 状态：✅ 成功 / ❌ 失败 / ⚠️ 部分成功
- 问题：如有

### 监控数据
- 错误率：
- 性能指标：

### 回滚方案
- 如有问题，执行：
```

## ⚠️ 红线
- 没有回滚方案不能发布
- 发布必须有人值守
- 监控异常必须立即响应
