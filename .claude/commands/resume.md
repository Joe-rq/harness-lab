# /resume — 恢复中断工作

读取当前项目状态，建议下一步行动。

## 执行步骤

1. 读取 `.claude/progress.txt` 获取当前 REQ 和阶段
2. 读取 `requirements/INDEX.md` 确认活跃 REQ 列表
3. 如果有 `.claude/.watchdog-state`，读取停滞/循环追踪数据
4. 读取最近一个 REQ 的当前内容（如有活跃 REQ）
5. 综合以上信息，输出状态摘要和建议

## 输出格式

```
📊 恢复状态

当前 REQ: [REQ 编号 + 标题]（或"无活跃 REQ"）
阶段: [当前阶段]
上次编辑: [最近编辑的文件/章节]

⚠️ 异常检测:
- [停滞/循环/风险信息，或"无异常"]

💡 建议下一步:
- [具体可执行的下一步行动]
```

## 异常情况处理

- **有活跃 REQ + 停滞**：建议参考 CLAUDE.md"异常响应协议"的停滞响应流程
- **有活跃 REQ + 循环**：建议参考 CLAUDE.md"异常响应协议"的循环响应流程
- **有活跃 REQ + R3+**：建议运行 `npm test` 验证
- **无活跃 REQ**：列出最近完成的 REQ，建议用户选择下一步任务
- **watchdog-state 不存在**：跳过异常检测部分，不影响其他信息

## 重要

- 只提供建议，不自动执行。用户确认后才行动。
- 如果发现 REQ 是 blocked 状态，明确告知并询问是否要解除阻塞。
