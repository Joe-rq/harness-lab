# REQ 模板填充时的章节重复问题

> 日期：2026-03-31
> 来源：REQ-2026-023 填充过程中发现并修复

## 问题

在填充 REQ 模板时，使用 `Edit` 工具修改某一章节，如果 `old_string` 末尾留有过多空行（或截断位置不当），会将下一个章节的标题行一并包含进 old_string 范围。

结果：
- 新字符串只写了目标章节的内容
- 下一个章节的标题仍留在文件中
- 导致文件中出现两段标题相同或相邻章节重复

## 案例

REQ-2026-023 模板中，"阻塞 / 搁置说明" 有两个相邻实例（第二个是原模板残留），原因是 Edit 时：

```
# 错误做法：old_string 截断位置不当
old_string = """
## 阻塞 / 搁置说明（可选）
- 原因：无
- 恢复条件：无
- 下一步：无

"""  # ← 这里多余的空行包含了下一章的标题行

new_string = """..."""  # 只替换了第一个章节
```

## 修复方式

发现问题后，使用 `Edit` 删除重复的章节：

```
old_string = """## 阻塞 / 搁置说明（可选）
- 原因：无
- 恢复条件：无
- 下一步：无

## 阻塞 / 搁置说明（可选）
- 原因：无
- 恢复条件：无
- 下一步：无
"""

new_string = """## 阻塞 / 搁置说明（可选）
- 原因：无
- 恢复条件：无
- 下一步：无
"""
```

## 预防建议

1. 填充 REQ 模板时，优先用 `Read` 确认当前文件状态，再决定编辑范围
2. Edit 时 `old_string` 应尽量精确匹配，**避免用多行空行作为截断点**
3. 替代方案：直接用 `Write` 重写整个文件，而不是多次 `Edit` 拼合
4. 可以在 `req:create` 后立即 `Read` 模板，观察章节结构再动手填充
