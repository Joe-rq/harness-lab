# Harness Lab × AutoResearch：精简优化方案

> 将 AutoResearch 的 **Scope Control** 和 **Results Logging** 理念引入 harness-lab，保持轻量，不做完整的自主实验循环。

---

## 1. 背景

### 1.1 AutoResearch 的两个精华设计

| 设计 | 说明 | 核心价值 |
|------|------|----------|
| **Scope Control** | 明确 Agent 能做什么、不能做什么 | 约束行为边界，降低失控风险 |
| **Results Logging** | 记录 baseline vs 改进后的对比 | 让改进效果可量化、可追溯 |

### 1.2 目标

只借鉴这两个设计，增强 harness-lab 的：
- Agent 行为可预期性
- REQ 完成效果可量化

---

## 2. Scope Control（约束字段）

### 2.1 问题

现有 REQ 的"范围"字段是描述性的，没有明确约束 Agent 的行为边界。

Agent 可能：
- 修改了不该改的文件
- 引入了未授权的依赖
- 做了超出 REQ 范围的事

### 2.2 方案

在 REQ 模板的"范围"部分，增加 **约束（Scope Control）** 子节：

```markdown
## 范围

### 涉及
- 目录 / 模块：
- 影响接口 / 页面 / 脚本：

### 约束（Scope Control）  ← 新增
> 明确 Agent 能做什么、不能做什么

**允许（CAN）**：
- 修改 `src/` 下的文件
- 添加 `tests/` 下的测试文件

**禁止（CANNOT）**：
- 修改 `prepare.py`
- 引入新依赖
- 改动 `docs/` 下的文档

**边界条件**：
- 单文件不超过 300 行
- 改动文件不超过 5 个
```

### 2.3 使用场景

| 场景 | 示例 |
|------|------|
| 自主 Agent 任务 | 明确 Agent 只能改 train.py |
| 多人协作 | 明确谁可以改哪些模块 |
| 敏感操作 | 禁止删除数据、禁止修改配置 |

## 3. Results Logging（结果记录）

### 3.1 问题

REQ 完成后，缺少"改进前后的指标对比"，无法证明改进效果。

### 3.2 方案

在 REQ 模板增加 **结果记录** 部分：

```markdown
## 结果记录  ← 新增

### Baseline
| 指标 | 值 | 说明 |
|------|-----|------|
| 响应时间 | 500ms | 改进前 |
| 内存占用 | 1.2GB | 改进前 |

### 改进后
| 指标 | 值 | 变化 |
|------|-----|------|
| 响应时间 | 200ms | -60% |
| 内存占用 | 0.8GB | -33% |

### 尝试记录
| 方案 | 指标结果 | 状态 | 说明 |
|------|----------|------|------|
| 方案A：增加缓存 | 400ms | discard | 提升不明显 |
| 方案B：算法优化 | 200ms | keep | 最终采用 |
```

### 3.3 填写时机

| REQ 类型 | 是否必需 |
|---------|---------|
| L2+ 性能/效果类 REQ | 建议填写 |
| L0/L1 快速修复 | 可选 |
| 文档类 REQ | 不适用 |

### 3.4 代理指标参考

如果 REQ 涉及效果改进但无直接指标，可使用代理指标：

| 工作类型 | 代理指标 |
|---------|---------|
| 性能优化 | 响应时间、内存占用、CPU 使用率 |
| 代码质量 | 测试覆盖率、lint 通过率 |
| 文档质量 | docs:verify 通过率、链接完整性 |
| 流程改进 | REQ 完成时间、review 周期 |

## 5. 不做的内容

以下内容**不在本次范围内**：

| 原方案内容 | 原因 |
|-----------|------|
| Level System（L0-L3） | 叠床架屋，现有字段已够用 |
| Structured Knowledge 目录重组 | 增量有限 |
| Meta-Skills 版本管理 | 可能适得其反 |
| 完整的自主实验循环 | 超出轻量治理定位 |

---

## 6. 兼容性

- 所有新增字段**可选填写**，不影响现有 REQ
- 现有 Skills 和目录结构**保持不变**
- 渐进采用，不强制迁移

---

## 7. 总结

| AutoResearch 设计 | harness-lab 对应 | 改进 |
|-------------------|------------------|------|
| Scope Control | REQ 约束字段 | 明确 Agent 行为边界 |
| Results Logging | REQ 结果记录 | 让改进效果可量化 |

核心原则：**轻量、增量、不添乱**。
