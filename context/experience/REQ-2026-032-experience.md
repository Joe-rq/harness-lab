# 2026-04-02 Experience 文档质量门禁

## 场景

REQ-2026-031 实现了"经验沉淀强制检查"，解决了从 0 到 1 的问题（确保有文档），但在元反思中发现了三个深层问题：
1. 命名约定与检查逻辑矛盾（日期命名文档无法被识别）
2. 存在性 ≠ 质量（空骨架可以通过检查）
3. 无内容校验（类似 REQ-2026-015 解决的空模板问题）

## 关联材料

- REQ: `requirements/in-progress/REQ-2026-032-experience-doc-quality-gate.md`
- Code Review: `requirements/reports/REQ-2026-032-code-review.md`
- QA: `requirements/reports/REQ-2026-032-qa.md`

## 问题 / 模式

- **软约束陷阱**：REQ-2026-031 的检查逻辑只看文件名前缀，19 篇日期命名文档被忽略
- **自举悖论**：空骨架文件可以过检查，违背"确保有价值经验被沉淀"的初衷
- **模式复用**：与 REQ-2026-015（空模板 REQ 阻断）完全相同的问题模式，只是对象不同

## 关键决策

- **决策 1：内容链接匹配**：除了文件名匹配，还支持读取文件内容查找 REQ 链接，兼容历史日期命名文档
- **决策 2：占位符检测**：复用 `req-validation.mjs` 的模式，定义 9 个关键占位符进行检测
- **决策 3：失败时明确提示**：列出未填充的章节，指导用户如何修复

## 解决方案

1. 在 `req-cli.mjs` 中添加 `findExperienceDocs` 函数，支持文件名和内容两种匹配方式
2. 添加 `validateExperienceContent` 函数，检测模板占位符
3. 修改 `completeCommand` 检查逻辑，先找文档再验内容
4. 更新 `context/experience/README.md` 明确命名约定为 `REQ-xxx-slug.md`

## 复用建议

- **内容校验模式**：定义占位符数组 → 遍历检测 → 返回问题列表，可用于任何模板内容验证
- **双重匹配策略**：文件名前缀 + 内容链接，兼顾新旧命名约定
- **错误提示设计**：列出具体问题 + 给出修复指导 + 提供豁免通道
