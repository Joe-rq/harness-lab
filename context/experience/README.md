## 🎯 沉淀原则

### 何时沉淀
1. 踩过坑后 → 立即记录
2. 解决难题后 → 总结方法
3. 发现模式后 → 提炼规范

### 如何沉淀
使用以下格式：

```markdown
# [标题]

## 场景
什么情况下会遇到

## 问题
具体表现是什么

## 原因
根本原因分析

## 解决方案
如何解决

## 预防措施
如何避免再次发生

## 相关链接
- 相关文档
- 相关代码

## 沉淀时间
YYYY-MM-DD
```

## 💡 示例

### 坑点示例
```markdown
# Apollo 配置格式坑点

## 场景
使用 Apollo 配置中心时

## 问题
配置读取失败，报错格式错误

## 原因
Apollo 的 JSON 配置必须是最外层是对象，不能是数组

## 解决方案
把数组包装在对象里：
```json
{
  "items": [...]
}
```

## 预防措施
- 配置模板中注释说明
- 代码中添加格式验证
- 文档中强调此约束

## 沉淀时间
2024-03-15
```
