# Product Overview

> 当前文件同时作为业务 context 最小示例，以及 `harness-lab` 仓库自身的产品概览。
> 复制到其他项目时，请替换成目标产品的真实事实。

## 产品目标

- 为持续迭代的软件项目提供一层稳定的研发治理协议
- 让人类开发者和 AI agent 能按同一套入口恢复上下文、推进 REQ、补齐验证和沉淀经验
- 把设计、评审、QA、发布从口头协作变成仓库内证据链

## 目标用户

- 维护长期产品仓库的工程团队
- 需要跨会话协作的人类开发者和 AI agent
- 希望把需求、设计、验证和经验沉淀放在同一仓库中的项目负责人

## 核心价值

- 固定 REQ 生命周期和交付物位置
- 用索引优先的方式减少上下文恢复成本
- 强制把 review / QA / ship 结果落盘
- 为后续项目沉淀可复用的经验文档

## 非目标

- 不提供 Web / App / Backend 的业务运行时框架
- 不替代目标项目自己的架构分层和技术选型
- 不替目标项目预设真实 `lint / test / build / verify` 命令

## 关键成功标准

- 新会话能在数分钟内定位当前 REQ 和状态
- 重要需求都有 REQ、设计稿和验证报告
- 接入项目会显式绑定自己的真实验证命令
- 已完成工作能回写为后续项目可复用的经验
